A Dialog With Myself

Umm....why do you want to make an ASP.NET MVC application that's as dynamic as possible?

It's all about trimming the fat. Getting rid of unnecessary code and capturing the essence of what you're trying to create:

How will we trim the statically typed fat without sacrificing maintainability?

Yep...that's what we're going to cover....let's start with a really bold statement. If you're not testing your ASP.NET MVC applications (dynamic or not), then you've doomed yourself and your fellow developers to an UNNECESSARY pile of legacy code.

Testing is important, I'm not going to deny that. But let's be realistic, we don't always have time to test.

Hopefully this post will help you find the time. We'll have more time to write tests if: we stop writing unnecessary code; take away unnecessary key strokes; stop unnecessary debugging; ease schema creation; and remove manual build and deploy steps. Making testing frictionless will make it easier to drive the implementation of the application using tests. You'll find that you're reliance on the compiler, intellisence, and static types just go away.

Alright. I'll keep an open mind about all this.
